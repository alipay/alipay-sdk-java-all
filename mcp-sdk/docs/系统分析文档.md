# Alipay MCP SDK 系统分析文档

## 一、项目背景

### 1.1 项目目标

Alipay MCP SDK 是支付宝开放平台提供的 Java SDK，基于 MCP (Model Context Protocol) 协议实现，使开发者能够便捷地调用支付宝开放平台的工具能力。

**核心目标：**
1. 提供纯 Java SDK，让任何 Java 应用能调用支付宝 MCP 工具
2. 提供 Spring Boot Starter，让 Spring AI 应用无缝集成支付宝工具

### 1.2 目标用户

| 用户类型 | 需求 | 推荐 SDK |
|----------|------|----------|
| 传统 Java 应用开发者 | 在应用中调用支付宝工具 | `alipay-mcp-sdk` |
| Spring AI 应用开发者 | 让 AI Agent 使用支付宝工具 | `alipay-mcp-spring-boot-starter` |

### 1.3 双 SDK 定位

```
┌─────────────────────────────────────────────────────────────────┐
│                        Alipay MCP 产品矩阵                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  SDK 1: alipay-mcp-sdk          SDK 2: alipay-mcp-spring-boot-  │
│  ────────────────────           starter                         │
│  纯 Java，无框架依赖              Spring AI 集成                   │
│                                                                 │
│  场景：传统 Java 应用              场景：Spring AI 应用             │
│  直接调用支付宝 MCP 工具            AI Agent 调用支付宝工具          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、核心流程

### 2.1 Client 模式流程（SDK 1）

```
┌─────────┐          ┌──────────────┐          ┌─────────────────┐
│ Client  │          │ SseConnection│          │ 支付宝 MCP 网关   │
└────┬────┘          └──────┬───────┘          └────────┬────────┘
     │                      │                           │
     │  connect(listener)   │                           │
     │─────────────────────►│                           │
     │                      │  GET /sse (SSE 连接)        │
     │                      │──────────────────────────►│
     │                      │                           │
     │                      │  event: endpoint          │
     │                      │◄──────────────────────────│
     │  onEndpointReady()   │                           │
     │◄─────────────────────│                           │
     │                      │                           │
     │  listTools()         │                           │
     │─────────────────────►│                           │
     │                      │  POST tools/list          │
     │                      │──────────────────────────►│
     │                      │  SSE event: message       │
     │                      │◄──────────────────────────│
     │  List<Tool>          │                           │
     │◄─────────────────────│                           │
     │                      │                           │
     │  callTool(name,args) │                           │
     │─────────────────────►│                           │
     │                      │  POST tools/call          │
     │                      │──────────────────────────►│
     │                      │  SSE event: message       │
     │                      │◄──────────────────────────│
     │  ToolCallResult      │                           │
     │◄─────────────────────│                           │
     │                      │                           │
```

**流程说明：**
1. **连接建立**：客户端调用 `connect()` 建立 SSE 长连接，等待服务端返回 endpoint 地址
2. **获取工具列表**：调用 `listTools()` 发送 HTTP POST 请求，响应通过 SSE 推回
3. **调用工具**：调用 `callTool()` 发送请求，等待 SSE 推送结果

3. **结果返回**：调用结果经 `ToolCallback` 返回给 AI Agent

| 配置项 | 必需 | 默认值 | 说明 |

### 4.3 拦截器说明

SDK 提供以下拦截器：

#### AlipayAuthInterceptor（支付宝认证签名拦截器）

**功能：** 生成 SHA256withRSA 签名并添加到请求头

**使用方式：** 按需添加，用户可根据业务场景选择是否使用

```java
McpClient client = new McpClientBuilder()
    .appId("2021006141677002")
    .privateKey("MIIEvgIBADAN...")
    .mcpName("my-client")
    .serverUrl("https://mcp-gateway.alipay.com")
    // 添加支付宝认证拦截器（按需）
    .addInterceptor(new AlipayAuthInterceptor(config))
    // 添加自定义日志拦截器
    .addInterceptor(chain -> {
        System.out.println(">>> Request: " + chain.request().getUrl());
        return chain.proceed(chain.request());
    })
    .build();
```

**拦截器执行顺序：** 按添加顺序依次执行

```
Request → AlipayAuthInterceptor → LoggingInterceptor → HTTP Client
```

#### 自定义拦截器

用户可实现 `Interceptor` 接口创建自定义拦截器：

```java
// 日志拦截器
Interceptor loggingInterceptor = chain -> {
    long start = System.currentTimeMillis();
    Request request = chain.request();
    System.out.println("[LOG] " + request.getMethod() + " " + request.getUrl());
    
    Response response = chain.proceed(request);
    
    long cost = System.currentTimeMillis() - start;
    System.out.println("[LOG] Response: " + response.getCode() + " (" + cost + "ms)");
    return response;
};

// 重试拦截器
Interceptor retryInterceptor = new Interceptor() {
    private static final int MAX_RETRY = 3;
    
    @Override
    public Response intercept(Chain chain) throws IOException {
        IOException lastException = null;
        for (int i = 0; i < MAX_RETRY; i++) {
            try {
                return chain.proceed(chain.request());
            } catch (IOException e) {
                lastException = e;
            }
        }
        throw lastException;
    }
};
```

## 五、SDK 2: alipay-mcp-spring-boot-starter 设计

### 5.1 核心思想

**让 Spring AI 应用中的 AI Agent 能够直接调用支付宝 MCP 工具。**

简单说就是：**把支付宝的 MCP 工具"注册"到 Spring AI 中，成为 AI Agent 可调用的 Tool。**

```java
/**
 * ToolCallback 提供者
 */
public class AlipayToolCallbackProvider implements ToolCallbackProvider {

    private final McpClient mcpClient;
    private final AlipayMcpServerProperties properties;
    private final Map<String, Tool> toolMap = new ConcurrentHashMap<>();

    @Override
    public ToolCallback[] getToolCallbacks() {
        // 1. 从支付宝 MCP 网关获取工具列表
        // 2. 根据配置过滤
        // 3. 转换为 Spring AI ToolCallback
        return toolMap.values().stream()
            .map(this::toToolCallback)
            .toArray(ToolCallback[]::new);
    }

    private ToolCallback toToolCallback(Tool alipayTool) {
        return ToolCallback.builder()
            .name(alipayTool.getName())
            .description(alipayTool.getDescription())
            .toolDefinition(parseToolDefinition(alipayTool))
            .toolCallback(context -> {
                // 调用支付宝 MCP 工具
                return mcpClient.callTool(
                    alipayTool.getName(), 
                    context.getArguments()
                );
            })
            .build();
    }
}
```

#### 2. AlipayToolCallbackProvider（工具回调提供者）

```java
public class AlipayToolCallbackProvider implements ToolCallbackProvider {

    private final McpClient mcpClient;
    private final AlipayMcpServerProperties properties;
    private final Map<String, Tool> toolMap = new ConcurrentHashMap<>();

    public AlipayToolCallbackProvider(McpClient mcpClient, AlipayMcpServerProperties properties) {
        this.mcpClient = mcpClient;
        this.properties = properties;
        this.initializeTools();  // 初始化时获取工具列表
    }

    private void initializeTools() {
        // 1. 调用支付宝 MCP 网关获取工具列表
        List<Tool> allTools = mcpClient.listTools();
        
        // 2. 根据配置过滤
        List<String> exposedTools = properties.getExposedTools();
        for (Tool tool : allTools) {
            if (exposedTools.isEmpty() || matchesPrefix(tool.getName(), exposedTools)) {
                toolMap.put(tool.getName(), tool);
            }
        }
    }

    @Override
    public ToolCallback[] getToolCallbacks() {
        // 3. 转换为 Spring AI ToolCallback
        return toolMap.values().stream()
            .map(this::toToolCallback)
            .toArray(ToolCallback[]::new);
    }

    private ToolCallback toToolCallback(Tool alipayTool) {
        return ToolCallback.builder()
            .name(alipayTool.getName())
            .description(alipayTool.getDescription())
            .toolDefinition(parseToolDefinition(alipayTool))
            .toolCallback(context -> {
                // 4. 实际调用支付宝工具
                return mcpClient.callTool(alipayTool.getName(), context.getArguments());
            })
            .build();
    }
}
```

**作用：**
- 启动时从支付宝 MCP 网关获取工具列表
- 将支付宝的 `Tool` 转换为 Spring AI 的 `ToolCallback`
- Spring AI Agent 启动时调用 `getToolCallbacks()` 获取所有可用工具

#### 3. 数据流转

```
┌─────────────┐
│ 支付宝 MCP   │
│ 网关        │
│             │
│ tools:      │
│ - invoice_1 │
│ - order_1   │
│ - pay_1     │
└──────┬──────┘
       │ listTools()
       ▼
┌─────────────────────────────┐
│ AlipayToolCallbackProvider  │
│                             │
│ toolMap: {invoice_1, ...}   │
└──────┬──────────────────────┘
       │ getToolCallbacks()
       │ 转换为 ToolCallback[]
       ▼
┌─────────────────────────────┐
│ Spring AI Agent             │
│                             │
│ 注册的工具：                 │
│ - invoice_1                 │
│ - order_1                   │
└──────┬──────────────────────┘
       │ AI 决定调用工具
       ▼
┌─────────────────────────────┐
│ ToolCallback.execute()      │
│                             │
│ → mcpClient.callTool()      │
└──────┬──────────────────────┘
       │
       ▼
┌─────────────┐
│ 支付宝 MCP   │
│ 网关        │
│ (执行并返回) │
└─────────────┘
```

### 5.5 配置说明

| 配置项 | 必需 | 默认值 | 说明 |
|--------|------|--------|------|
| `alipay.mcp.app-id` | 是 | - | 应用 ID |
| `alipay.mcp.private-key` | 是 | - | 私钥 (PKCS8) |
| `alipay.mcp.mcp-name` | 是 | - | MCP 服务名称 |
| `alipay.mcp.spring-ai.enabled` | 否 | `true` | 启用 Spring AI 集成 |
| `alipay.mcp.exposed-tools` | 否 | `[]` | 要注册的工具前缀列表（为空则全部注册） |

## 六、安全设计

### 6.1 签名机制

采用 SHA256withRSA 签名算法，签名原文格式：

```
{authString}
{method}
{url}
{body}
```

authString 格式：`app_id={appId},timestamp={timestamp},nonce={nonce}`

Authorization Header：`ALIPAY-SHA256withRSA {authString},sign={sign}`

### 6.2 工具过滤配置

```yaml
alipay:
  mcp:
    # 只注册发票和订单相关工具
    exposed-tools:
      - invoice_
      - order_
```
